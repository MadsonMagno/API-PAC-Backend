# ⚙️ API PAC - Backend de Gestão de Pacientes

[![Laravel](https://img.shields.io/badge/Laravel-Framework-red.svg)](https://laravel.com/)
[![API](https://img.shields.io/badge/Status-RESTful%20API-blue.svg)]()
[![HealthTech](https://img.shields.io/badge/Dom%C3%ADnio-HealthTech-blue.svg)]()

Esta é a API de backend robusta projetada para servir o sistema PAC (Pronto Atendimento/Controle), gerenciando a lógica de negócios, autenticação e persistência de dados clínicos.

## 🚀 Funcionalidades

- **Endpoints RESTful:** Gerenciamento completo de fluxos de pacientes e agendamentos.
- **Segurança de Dados:** Implementação de middlewares de autenticação e validação rigorosa.
- **Integração:** Projetada para comunicação *headless* com o frontend `pac`.
- **Performance:** Otimizada para consultas em bancos de dados de alta volumetria (Oracle/MySQL).

---

## 🛠️ Stack Técnica

- **Linguagem:** PHP 8.x
- **Framework:** Laravel
- **Banco de Dados:** Compatível com Eloquent ORM.
- **Testes:** Automatizados via PHPUnit.

---

## 🚦 Como Iniciar

1. **Instalar:** `composer install`
2. **Setup:** `cp .env.example .env && php artisan key:generate`
3. **Servidor:** `php artisan serve`
